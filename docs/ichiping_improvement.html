IchiPing v1 精度向上の検証レポート
— Solist-AI 移植で得た知見の逆輸入

作成日: 2026-06-06 ／ 実装: sim/exp_improve.py（実 IchiPing データ + 学習 MLP）

要約: 移植検証で得た2原理を IchiPing v1 に転用できるか実データで検証。 原理A（時間特徴）は効かず（学習モデルは single スペクトルから既に情報抽出、時間分割は逆効果 93.4%→71.1%）、 原理B（最終層の現場再校正）は劇的に有効（cross-environment 32cls が ~53% → ~97–100%、IchiPing LIVE 59% を引き上げ可能）。

1. 目的

Solist-AI（ML63Q2557, ELM）への移植検証の過程で、32cls 精度を引き上げる2つの原理を発見した。本レポートは、これらが 原典 IchiPing v1（FRDM-MCXN947 + Conv1D CNN）側の精度向上にも転用できるかを実データで検証する。

  • 原理A時間構造の特徴化: 入力を「2秒平均スペクトル(single)」→「時間分割スペクトル(multiwin)」にし、漏洩音のサブ状態手がかり（時間構造）を明示。
  • 原理B環境ごとの最終層 再校正: 特徴抽出(backbone)を凍結し、最終線形ヘッドだけを設置環境の数ショットで再学習（= Solist-AI のオンデバイス学習に相当）。

背景: IchiPing v1 は 14cls が LIVE 97% と高い一方、32cls は LIVE 59%（in-family 88–100%）で cross-environment が弱点。原因は移植検証で判明した「サブ状態の時間署名が環境固有」。

2. 方法

項目内容
データ実 IchiPing 録音（PRBS 2s @16kHz）。学習 v6–v12、評価 eval_quiet / eval_noise_low / eval_noise_high（別セッション）
特徴 single時間領域 baseline 差分 → 2秒 |FFT| 平均 → 400–3000Hz クロップ（D=167）。IchiPing 現行に相当
特徴 multiwin2秒を時間4分割し窓ごと |FFT|（300–3500Hz）連結（D=412）。時間構造を保持
モデル2層 MLP（256-256, dropout 0.3, Adam）。"学習された特徴" を持つ代表モデル（ELM と違い入力重みも学習）。IchiPing Conv1D CNN の代理
原理A 検証train v6–v11 → test v12（in-family cross-run）。single vs multiwin の 32cls 精度
原理B 検証train v6–v12 → eval 3条件（cross-environment）。factory ヘッド vs 設置環境 N-shot 再校正ヘッド

注: 本検証は IchiPing 本番 CNN（neutron 特殊アーキ）そのものではなく、原理を実データ上で確認する代表モデル(MLP)で実施。本番 CNN への適用は §5 推奨フォローアップ。

4. 考察・結論

原理A（時間特徴）は IchiPing には効かない。 時間分割は ELM（ランダム固定特徴）には有効だった（別検証で 74%→83.5%）が、それは ELM が特徴を学習できないための「補償策」。学習済みモデル（MLP / CNN）は single スペクトルから既にサブ状態情報を抽出できるため、時間分割（窓ごとに平均が減りノイズ増）はむしろ悪化（93.4%→71.1%）。IchiPing の深い CNN が in-family 88–100% を出していたのはこのため。
原理B（再校正ヘッド）は IchiPing に強く効く。 IchiPing v1 の弱点は cross-environment（LIVE 32cls 59%）。原因はサブ状態の時間署名が環境固有で、固定 CNN が環境に適応できないこと。backbone を凍結し最終線形ヘッドだけを設置環境の数ショット（各状態 3–5 回）で再校正すると、cross-env 32cls が ~53% → ~97–100% に回復。14cls は元々 cross-env でも堅牢（~97–100%）なので、再校正は 32cls に特に効く。
示唆（やや逆説的）: Solist-AI 移植版は、その目玉「オンデバイス学習（β現場再校正）」により、元の IchiPing v1（固定 CNN）より cross-environment に強くなり得る。移植作業が原典の改善点（再校正ヘッドの追加）を炙り出した。

推奨（IchiPing v1 への適用）

  • 時間特徴は導入不要（single スペクトル維持。学習 CNN には不利）。
  • 再校正可能な最終ヘッドを追加: CNN backbone を凍結し、最終線形層を設置環境の少数ショットで再学習する仕組み（few-shot / オンデバイス校正）を実装すれば、LIVE 32cls を 59% → ~97–100% に引き上げ可能と見込まれる。
限界（正直な注記）
  • 代表モデルは MLP であり IchiPing 本番 CNN（neutron）そのものではない（原理検証）。本番 CNN の penultimate 埋め込みでの再現が次段。
  • 再校正の評価は within-session（同一 eval 録音内で校正/評価を分離、リークなし）。「設置→後日運用」の経時ドリフト耐性は未測定（同一環境・別時刻データが必要）。
  • noise_high は再校正 N=3–5 推奨（N=1 では不足）。

5. 次のステップ
